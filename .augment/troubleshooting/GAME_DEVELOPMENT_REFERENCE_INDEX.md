## 🔍 快速問題診斷

### 症狀 1：遊戲對象顯示在錯誤的位置

**可能原因**：
- [ ] 對象被添加到容器後沒有設置本地座標
- [ ] 容器的原點設置不正確
- [ ] 座標轉換公式計算錯誤

**解決步驟**：
1. 查看 `PHASER_CONTAINER_COORDINATE_SYSTEM_GUIDE.md` - 場景 1 或 2
2. 驗證本地座標是否被正確設置
3. 使用調試技巧打印座標信息

### 症狀 2：頁面導航後遊戲狀態沒有恢復

**可能原因**：
- [ ] 沒有保存所有相關的狀態信息
- [ ] 恢復邏輯有條件地跳過了某些狀態
- [ ] 視覺指示器沒有被恢復

**解決步驟**：
1. 查看 `GAME_ISSUES_AND_SOLUTIONS_REFERENCE.md` - 問題 2 和 3
2. 檢查保存邏輯是否完整
3. 檢查恢復邏輯是否完整

### 症狀 3：調試時無法確定對象的位置

**解決步驟**：
1. 查看 `PHASER_CONTAINER_COORDINATE_SYSTEM_GUIDE.md` - 調試技巧
2. 使用 `printCoordinates()` 函數打印座標信息
3. 使用視覺調試添加標記

## 📋 使用流程

### 開發新遊戲時

1. **第一步**：閱讀 `GAME_ISSUES_AND_SOLUTIONS_REFERENCE.md`
   - 了解常見問題
   - 學習最佳實踐

2. **第二步**：根據需要查看 `PHASER_CONTAINER_COORDINATE_SYSTEM_GUIDE.md`
   - 如果使用容器
   - 如果需要保存/恢復狀態

3. **第三步**：遇到問題時
   - 使用快速問題診斷
   - 查看相應的文檔部分
   - 參考 Match-up Game 的實現

### 修復遊戲 Bug 時

1. **診斷問題**：使用快速問題診斷
2. **查找解決方案**：在相應文檔中查找
3. **參考實現**：查看 Match-up Game 的代碼
4. **驗證修復**：使用調試技巧驗證

## 💡 最佳實踐總結

### 容器使用

✅ **DO**：
- 在添加到容器後設置本地座標
- 計算相對座標 = 對象座標 - 容器座標
- 使用調試工具驗證座標

❌ **DON'T**：
- 忘記設置本地座標
- 假設對象會自動定位
- 混淆世界座標和本地座標

### 狀態管理

✅ **DO**：
- 保存所有相關的狀態信息
- 完整恢復所有狀態
- 驗證恢復後的狀態

❌ **DON'T**：
- 有條件地跳過某些狀態
- 只保存部分狀態
- 忘記恢復視覺指示器
