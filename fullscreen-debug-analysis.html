🔍 shimozurdo 全螢幕黑色區域問題診斷
深度分析全螢幕功能失效的根本原因

🚨 問題根源分析

  • ❌

    CSS 樣式限制問題

    HTML 中的 #game 容器沒有設定全螢幕樣式，導致在全螢幕模式下無法填滿整個螢幕空間。

    /* 當前 CSS - 缺少全螢幕樣式 */ body { background: #000; overflow: hidden; } /* 缺少 #game 容器的全螢幕樣式 */
  • ⚠️

    Phaser Scale 配置不完整

    main.js 中缺少全螢幕相關的 scale 配置，沒有設定 fullscreenTarget 和相關參數。

    // 當前配置 - 缺少全螢幕設定 scale: { mode: Phaser.Scale.RESIZE, parent: 'game', // 缺少 fullscreenTarget 等設定 }
  • 🔧

    全螢幕目標元素問題

    utils/screen.js 中設定的 fullscreenTarget 為 #game 元素，但該元素沒有適當的全螢幕樣式。

    // utils/screen.js this.scale.fullscreenTarget = document.getElementById('game') // 但 #game 元素缺少全螢幕 CSS
  • 📱

    響應式適配問題

    全螢幕模式下，RESIZE scale mode 沒有正確處理容器尺寸變化，導致黑色區域出現。

✅ 完整解決方案

  • 步驟 1：修復 CSS 樣式

    在 index.html 中添加全螢幕 CSS 樣式：

    #game { width: 100%; height: 100%; } #game:fullscreen { width: 100vw !important; height: 100vh !important; } #game:-webkit-full-screen { width: 100vw !important; height: 100vh !important; } #game:-moz-full-screen { width: 100vw !important; height: 100vh !important; }

  • 步驟 2：完善 Phaser Scale 配置

    在 main.js 中添加全螢幕相關配置：

    scale: { mode: Phaser.Scale.RESIZE, parent: 'game', width: SIZE_WIDTH_SCREEN, height: SIZE_HEIGHT_SCREEN, fullscreenTarget: 'game', expandParent: true, autoCenter: Phaser.Scale.CENTER_BOTH, // ... 其他配置 }

  • 步驟 3：優化全螢幕事件處理

    改進 utils/screen.js 中的全螢幕實現：

    function fullScreen() { this.scale.fullscreenTarget = document.getElementById('game') // 添加全螢幕狀態變化監聽 this.scale.on('fullscreenchange', () => { if (this.scale.isFullscreen) { // 全螢幕模式下的特殊處理 this.scale.refresh() } }) // ... F11 鍵處理 }

  • 步驟 4：添加響應式刷新

    在全螢幕切換時強制刷新響應式佈局：

    // 在 Handler Scene 中添加 updateFullscreenResize() { if (this.scale.isFullscreen) { // 強制重新計算尺寸 this.scale.refresh() // 更新所有場景的相機 this.scene.scenes.forEach(scene => { if (scene.cameras && scene.cameras.main) { this.updateCamera(scene) } }) } }
