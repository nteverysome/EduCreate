# Phase 4 - Step 2：性能測試計劃

## 📋 測試目標

1. **首屏加載時間** - 測試遊戲初始化速度
2. **渲染性能** - 測試 FPS 和幀時間
3. **內存使用** - 測試內存佔用和洩漏
4. **FPS 穩定性** - 測試遊戲運行的穩定性

## 🎯 測試指標

### 首屏加載時間
- **目標**: < 3 秒
- **測試方法**: 使用 Playwright 測量從頁面加載到遊戲可交互的時間
- **測試設備**: Desktop (1280×800)

### 渲染性能
- **目標**: 平均 FPS ≥ 55 (60 FPS 的 90%)
- **測試方法**: 使用 Chrome DevTools Performance API
- **測試場景**: 
  - 遊戲初始化
  - 卡片拖曳
  - 卡片匹配

### 內存使用
- **目標**: 初始內存 < 50MB，峰值 < 100MB
- **測試方法**: 使用 Chrome DevTools Memory API
- **測試場景**:
  - 遊戲初始化
  - 遊戲運行 30 秒
  - 卡片交互

### FPS 穩定性
- **目標**: FPS 波動 < 10%
- **測試方法**: 監控 Phaser 的 FPS 計數器
- **測試時間**: 60 秒

## 📊 測試設備

| 設備 | 解析度 | 優先級 |
|------|--------|--------|
| Desktop | 1280×800 | 高 |
| iPad mini | 768×1024 | 中 |
| iPhone 12 | 390×844 | 中 |

## 🔧 測試工具

1. **Playwright** - 自動化測試框架
2. **Chrome DevTools** - 性能分析
3. **Phaser 內置工具** - FPS 監控
