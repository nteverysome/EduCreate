## 📐 計算示例對比

### 場景 1：iPhone 14 直向（390×844px）- 5列 4行

#### Wordwall 方式
```
卡片寬度 = 390 / 5 = 78px
卡片高度 = 844 / 4 = 211px

英文字大小 = 211 × 0.35 = 73.85px
中文字大小 = 211 × 0.25 = 52.75px

totalUnitHeight = 211 + 52.75 = 263.75px
maxRows = 844 / 263.75 = 3.2 → 3 行
```

#### EduCreate v7.0 方式
```
卡片寬度 = 390 / 5 = 78px
卡片高度 = 211px

中文字高度 = 211 × 0.4 = 84.4px

totalUnitHeight = 211 + 84.4 + 3 = 298.4px
maxRows = 844 / 298.4 = 2.8 → 2 行
```

#### 對比
```
Wordwall：
- 可顯示 3 × 5 = 15 個卡片
- 中文字大小：52.75px
- 視覺效果：緊湊

EduCreate v7.0：
- 可顯示 2 × 5 = 10 個卡片
- 中文字大小：84.4px
- 視覺效果：寬鬆
```

#### 缺點 ❌

---

### 混合方式（推薦）

#### 計算方式
```javascript
let textHeightRatio = 0.4;  // 默認 40%

if (finalCardHeight < 80) {
    textHeightRatio = 0.3;  // 小卡片：30%
} else if (finalCardHeight > 150) {
    textHeightRatio = 0.35;  // 大卡片：35%
}

const chineseTextHeight = finalCardHeight * textHeightRatio;
```

#### 優點 ✅
- 自動適應卡片大小
- 小屏幕時更緊湊
- 大屏幕時更易讀
- 最佳用戶體驗

#### 缺點 ❌
- 計算稍複雜
- 需要測試多種場景

## 📝 決定因素

### Wordwall 使用 0.25-0.3 的原因

1. **遊戲設計**
   - 強調快速反應
   - 需要更多卡片
   - 視覺刺激

2. **屏幕利用率**
   - 最大化卡片數量
   - 減少滾動
   - 一屏顯示更多

3. **用戶體驗**
   - 快速遊戲節奏
   - 視覺豐富
   - 挑戰性高

### EduCreate 使用 0.4 的原因

1. **教育設計**
   - 強調易讀性
   - 中文字更大
   - 學習友好

2. **可讀性**
   - 減少眼睛疲勞
   - 適合長時間學習
   - 符合中文習慣

3. **用戶體驗**
   - 舒適的學習環境
   - 清晰的文字
   - 專注於內容
