## 🎯 中文字大小決定方式

### 核心原則

**Wordwall 使用基於卡片大小的相對字體大小**

### 計算方式

#### 1. 卡片尺寸計算
```
卡片寬度 = Canvas寬度 / 列數
卡片高度 = Canvas高度 / 行數

例如：
卡片寬度 = 888 / 5 = 177.6px
卡片高度 = 515 / 4 = 128.75px
```

#### 2. 中文字大小計算
```
中文字大小 = 卡片高度 × 比例

根據觀察：
- 英文字大小：卡片高度 × 0.35-0.4
- 中文字大小：卡片高度 × 0.25-0.3

例如：
英文字大小 = 128.75 × 0.35 ≈ 45px
中文字大小 = 128.75 × 0.25 ≈ 32px
```

#### 3. 字體選擇
```
英文字體：Open Sans, Rubik, BalooBhaijaan, Arial, sans-serif
中文字體：同上（系統會自動選擇支持中文的字體）
```

## 🔍 Wordwall 的設計特點

### 1. 響應式設計
- Canvas 寬度和高度根據屏幕大小自動調整
- 卡片大小根據 Canvas 大小自動計算
- 字體大小根據卡片大小自動調整

### 2. 相對字體大小
- 不使用固定像素值
- 使用卡片高度的百分比
- 自動適應不同屏幕

### 3. 多語言支持
- 英文和中文使用不同的字體大小比例
- 中文字體通常比英文小 20-30%
- 確保中文字不會超出卡片邊界

### 4. 視覺平衡
- 英文字大小：卡片高度 × 0.35-0.4
- 中文字大小：卡片高度 × 0.25-0.3
- 中文字通常在英文字下方，不會重疊

## 🎯 建議

### 對 EduCreate 的建議

#### 方案 1：採用 Wordwall 的比例
```javascript
// 英文字大小
const englishFontSize = finalCardHeight * 0.35;

// 中文字高度
const chineseTextHeight = finalCardHeight * 0.25;
```

**優點**：
- 與 Wordwall 一致
- 更緊湊的佈局
- 更多卡片可以顯示

**缺點**：
- 中文字可能太小
- 可讀性降低

#### 方案 2：保持 EduCreate v7.0 的比例
```javascript
const chineseTextHeight = finalCardHeight * 0.4;
```

**優點**：
- 中文字更大
- 易於閱讀
- 符合中文閱讀習慣

**缺點**：
- 需要更多垂直空間
- 可能需要分頁

#### 方案 3：混合方案（推薦）
```javascript
// 根據卡片大小動態調整
let textHeightRatio = 0.4;  // 默認 40%

if (finalCardHeight < 80) {
    textHeightRatio = 0.3;  // 小卡片：30%（Wordwall 風格）
} else if (finalCardHeight > 150) {
    textHeightRatio = 0.35;  // 大卡片：35%
}

const chineseTextHeight = finalCardHeight * textHeightRatio;
```

**優點**：
- 自動適應卡片大小
- 小屏幕時更緊湊
- 大屏幕時更易讀
- 最佳的用戶體驗

## 📝 核心發現

### Wordwall 的決定因素

1. **Canvas 尺寸**
   - 根據屏幕大小自動調整
   - 響應式設計

2. **卡片數量**
   - 決定行數
   - 影響卡片高度

3. **卡片高度**
   - 決定字體大小
   - 相對比例：0.25-0.4

4. **語言類型**
   - 英文：0.35-0.4
   - 中文：0.25-0.3
