# ❓ 語言卡片系統常見問題解答

## 🎯 基本概念

### Q1：什麼是語言卡片？
**A：** 語言卡片是遊戲中顯示英文-中文詞彙對應的卡片。包括：
- 中文卡片（左側）
- 英文卡片（右側）
- 音標和發音
- 詞性和難度標記

### Q2：語言卡片和活卡片有什麼區別？
**A：** 
- **語言卡片**：顯示詞彙內容（英文-中文）
- **活卡片**：顯示遊戲狀態（分數、進度）

### Q3：為什麼需要雙語系統？
**A：** 為了幫助用戶學習英文：
- 中文提示幫助理解
- 英文發音幫助發音
- 雙語結合提升學習效果

---

## 🔧 技術問題

### Q4：BilingualManager 是什麼？
**A：** 管理語音播放的模塊：
- 播放中文發音
- 播放英文發音
- 控制播放順序和速度

### Q5：GEPTManager 是什麼？
**A：** 管理詞彙等級的模塊：
- 按等級加載詞彙
- 管理詞彙難度
- 支持自定義詞彙

### Q6：詞彙是如何加載的？
**A：** 三層加載流程：
```
1. 遊戲入口頁面 → 驗證 activityId
2. GameSwitcher → 生成 URL 並注入參數
3. iframe 內部 → 解析參數並加載詞彙
```

## 📱 手機版問題

### Q7：為什麼手機版卡片顯示不好？
**A：** 主要原因：
- 卡片尺寸固定，未根據屏幕適配
- 文字大小不合理
- 觸摸區域過小

### Q8：如何修復手機版卡片？
**A：** 三個步驟：
1. 實現響應式卡片尺寸
2. 調整文字大小
3. 擴大觸摸區域

### Q9：手機版最小觸摸目標是多少？
**A：** 根據 WCAG 標準，最小應為 **44x44px**

## ⚡ 性能問題

### Q10：為什麼詞彙加載慢？
**A：** 主要原因：
- 每次都調用 API
- 缺少緩存機制
- 網絡延遲

### Q11：如何加快詞彙加載？
**A：** 使用 localStorage 緩存：
```javascript
const cached = localStorage.getItem(`vocab_${id}`);
if (cached) return JSON.parse(cached);
```

### Q12：內存占用太高怎麼辦？
**A：** 優化方案：
- 圖片懶加載
- 虛擬滾動
- 語音流式播放

## 🔊 語音問題

### Q13：為什麼語音播放不穩定？
**A：** 主要原因：
- 瀏覽器 Speech API 支持不一致
- 設備語言設置不同
- 語音隊列管理不當

### Q14：如何改進語音播放？
**A：** 改進方案：
- 添加錯誤處理
- 實現語音隊列
- 提供文字備選

### Q15：支持哪些語言的語音？
**A：** 目前支持：
- 中文（普通話）
- 英文（美式/英式）
- 可擴展支持其他語言

## 🎮 遊戲相關

### Q16：語言卡片如何在 Match-Up Game 中使用？
**A：** 
- 左側顯示中文卡片
- 右側顯示英文卡片
- 用戶拖放進行配對

### Q17：如何自定義詞彙？
**A：** 通過詞彙管理器：
1. 進入詞彙管理頁面
2. 創建新詞彙集
3. 添加英文-中文對應
4. 選擇 GEPT 等級

### Q18：如何切換 GEPT 等級？
**A：** 在遊戲頂部選擇：
- Elementary（初級）
- Intermediate（中級）
- Advanced（高級）

---

## 📊 數據相關

### Q19：詞彙數據存儲在哪裡？
**A：** 三個位置：
- 雲端：API 服務器
- 本地：localStorage
- 內存：JavaScript 變量

### Q20：如何同步詞彙數據？
**A：** 同步流程：
1. 從雲端加載
2. 存儲到 localStorage
3. 加載到內存
4. 顯示在遊戲中

## 🚀 改進相關

### Q21：改進需要多長時間？
**A：** 根據優先級：
- 高優先級：3-4 天
- 中優先級：2-3 天
- 低優先級：1-2 天

### Q22：改進會影響現有功能嗎？
**A：** 不會，改進方案：
- 向後兼容
- 漸進式改進
- 灰度發布

### Q23：如何驗證改進效果？
**A：** 使用以下指標：
- 加載時間 < 1.5s
- 內存占用 < 30MB
- 用戶滿意度 > 4.5/5
