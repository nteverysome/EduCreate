# Wordwall 中文字大小決定方式 - 最終總結

## 🎯 核心發現

### Wordwall 的中文字大小決定方式

**基於卡片高度的相對比例**

```
中文字大小 = 卡片高度 × 0.25-0.3
```

---

## 📊 詳細分析

### 1. Canvas 尺寸
```
Canvas 寬度：888px
Canvas 高度：515px
```

### 2. 卡片佈局
```
5 列 × 4 行 = 20 個卡片
卡片寬度 = 888 / 5 = 177.6px
卡片高度 = 515 / 4 = 128.75px
```

### 3. 字體大小
```
英文字大小 = 128.75 × 0.35 ≈ 45px
中文字大小 = 128.75 × 0.25 ≈ 32px
```

### 4. 決定因素

#### 主要因素
1. **Canvas 尺寸**
   - 根據屏幕大小自動調整
   - 響應式設計

2. **卡片數量**
   - 決定行數
   - 影響卡片高度

3. **卡片高度**
   - 決定字體大小
   - 相對比例計算

#### 次要因素
1. **語言類型**
   - 英文：0.35-0.4
   - 中文：0.25-0.3

2. **字體族**
   - Open Sans, Rubik, BalooBhaijaan, Arial, sans-serif
   - 系統自動選擇支持中文的字體

---

## 🔄 計算流程

```
屏幕寬度和高度
    ↓
Canvas 寬度和高度
    ↓
卡片數量（列數和行數）
    ↓
卡片寬度 = Canvas寬度 / 列數
卡片高度 = Canvas高度 / 行數
    ↓
英文字大小 = 卡片高度 × 0.35-0.4
中文字大小 = 卡片高度 × 0.25-0.3
    ↓
渲染到 Canvas
```

#### 方案 3：混合方案（推薦用於通用）
```javascript

    textHeightRatio = 0.3;  // 小卡片

    textHeightRatio = 0.35;  // 大卡片
}

const chineseTextHeight = finalCardHeight * textHeightRatio;
```

## 🔍 關鍵洞察

### 1. 響應式設計
- Wordwall 完全響應式
- 所有尺寸都是相對的
- 自動適應不同屏幕

### 2. 相對字體大小
- 不使用固定像素值
- 基於卡片高度的百分比
- 自動縮放

### 3. 多語言支持
- 英文和中文使用不同比例
- 中文字通常比英文小 20-30%
- 確保不超出邊界

### 4. 視覺平衡
- 英文字在上方
- 中文字在下方
- 不會重疊
